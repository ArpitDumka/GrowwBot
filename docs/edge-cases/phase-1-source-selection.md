# Phase 1 — Edge Cases: Source Selection & Corpus Lock

> Companion to `../architecture.md` Phase 1. Corpus is **locked to exactly the 10 Groww URLs** in `phase-1/config/sources.yaml`.

## Edge cases

| # | ID | Trigger | Expected behavior | Detection | Mitigation | Priority |
|---|---|---|---|---|---|---|
| 1 | 1.01 | Groww renames a scheme slug (e.g. `hdfc-equity-fund-direct-growth` → `hdfc-flexi-cap-fund-direct-growth`); old URL returns 301 | Fetcher follows the redirect, but logs a `URL_CHANGED` event so the team can update `sources.yaml`; the snapshot is stored under the new canonical URL | Fetcher logs `final_url != configured_url` | Allow up to 1 redirect hop; raise CI alert when detected so the YAML is updated in the next PR | P0 |
| 2 | 1.02 | Groww renames a slug and returns 404 on the old URL | Build fails loudly; that scheme is not silently dropped | Ingest exits non-zero; missing `source_id` from index manifest | Treat any 404 from a configured URL as a build-blocking error | P0 |
| 3 | 1.03 | A scheme in the corpus is merged or wound up by the AMC; Groww shows a "scheme is closed" notice | Page is still ingested but tagged `status = closed`; intent layer routes any query about it to a polite "this scheme is no longer accepting investments — refer to the linked page" response | Section extractor finds the "closed" banner | Add a `closed_scheme` template; gate retrieval to show it before any factual answer | P1 |
| 4 | 1.04 | Two schemes share a confusable name (e.g. *HDFC Defence Fund* vs a hypothetical *HDFC Defence ETF*) | Alias dictionary disambiguates; if ambiguous, refuse and ask the user to clarify | Multiple aliases match the same query token | Mark alias entries with `ambiguous_with: [...]` and force a clarification prompt | P1 |
| 5 | 1.05 | User asks about Direct Plan but corpus only has Regular Plan (or vice versa) | Answer uses the **Direct Growth** numbers we have and explicitly states that in the response (e.g., "for the Direct Plan…") | Compose-time check: append "Direct Plan" qualifier when the chunk's `scheme` ends with "Direct Growth"/"Direct Plan Growth" | Hard-code the plan qualifier into the answer template per scheme | P0 |
| 6 | 1.06 | Groww page shows the AMC's **total AUM** (e.g. ₹9,37,047 Cr) instead of the scheme's AUM | Field detector must extract the **scheme-level** AUM from the header card, not the AMC AUM in the "Fund house" block | Sanity check: scheme AUM should differ from AMC AUM (unless scheme = AMC, never the case here) | Whitelisted DOM selector for header AUM; reject any AUM ≥ 5,00,000 Cr as "AMC AUM, not scheme AUM" | P0 |
| 7 | 1.07 | ELSS lock-in is a **statutory** rule (3 years) but not always written in the page text | Pre-seed `hdfc_elss` chunks with a deterministic `lock_in: 3 years (statutory ELSS)` field, marked `source: SEBI ELSS rules` in the chunk metadata | Field detector returns empty `lock_in` for `category = ELSS` | Inject a constant chunk during indexing for ELSS schemes only | P0 |
| 8 | 1.08 | Special characters in scheme names break alias matching (e.g., `HDFC Pharma & Healthcare Fund`) | Alias normalizer strips/normalizes `&` → `and`, smart quotes → straight, NFC-normalizes Unicode | Alias-lookup misses on a query that contains the scheme name verbatim | Normalize both sides of the comparison before matching | P1 |
| 9 | 1.09 | Refresh frequency for NAV (daily) ≠ refresh for static fields (rare) | All 10 URLs are re-fetched nightly regardless; the chunker only re-embeds chunks whose **content hash** changed (NAV chunks change daily, exit-load chunks rarely) | Compare per-chunk hash, not per-document hash | Per-chunk content-hash diff in `build_index.py` | P1 |
| 10 | 1.10 | Aliases collide (e.g., "hdfc equity" historically referred to *HDFC Flexi Cap Fund*) | The legacy alias maps to the current canonical scheme; canonical is logged for audit | Alias map test asserts each surface form maps to exactly one scheme | Unit test on `aliases.yaml` enforcing `surface_form → 1 scheme` | P0 |
| 11 | 1.11 | User specifies "HDFC Equity Fund" expecting a fund that no longer exists under that name | Treat as alias for *HDFC Flexi Cap Fund* and proceed; answer footer can mention "(formerly HDFC Equity Fund)" once if the alias was used | Alias hit on a `legacy_name` entry | Mark some aliases with `legacy: true` so the response can prepend the rename note | P2 |
| 12 | 1.12 | Corpus drift: someone adds an 11th URL to `sources.yaml` without updating allow-list logic in Phase 7 | CI fails because the citation allow-list test sees a URL count mismatch | Test: `len(sources.yaml) == len(citation_allowlist)` | Single source of truth: generate the allow-list from `sources.yaml` at build time | P0 |
